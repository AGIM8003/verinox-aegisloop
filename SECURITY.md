# Security and Responsible Disclosure

This release is an architecture and publication package, not a deployed system. Requirements and tests do not represent implemented controls.

If executable components are later published and a vulnerability or unsafe behavior is found:

1. do not post credentials, personal data, actuator secrets, exploit details, or operational topology publicly;
2. use GitHub private security advisories when available;
3. otherwise contact the repository owner through the account's private published channel;
4. identify the version, component, safety impact, reproduction conditions, and proposed mitigation; and
5. allow reasonable triage time before public disclosure when physical or user safety could be affected.

Prior-art references, architecture criticism, reproducibility questions, and documentation corrections may be submitted publicly if they contain no sensitive data.

No bounty, response-time commitment, safety certification, or warranty is offered.
